The Journey

What actually happened

These are the real sessions. The real errors. Not repositioned as "learning opportunities" — just what broke, what the error said, and what it took to fix it.

Early 2026 — Session Zero
No map. Starting anyway.

Started with a blank machine and a clear outcome: build a live AI-powered web application. No tutorials, no bootcamp. Direct to production. The first thing I tried to do was navigate to a folder.

bash: cd: No such file or directory
cd C:\Users\phill\Documents\GitHub\velocity-architecture

broke Pasted a Windows path into Git Bash. Git Bash doesn't speak Windows.

learned C:\path becomes /c/path in Git Bash. One rule. Applies every time, forever.

Phase 1 — VAF Agentic Architect
Core agent: orchestrator, agents, KB loader, action engine.

The app reads the VAF knowledge base from GitHub, sends a topic to Claude, gets back an enterprise architecture artefact, commits it to the repo. Built in TypeScript. First real thing I'd directed being built. Then tried to push it.

error: src refspec main does not match any

broke Tried git push -u origin main but the branch was called master.

learned Newer GitHub defaults to main. Older local git defaults to master. They don't tell you when they disagree. Check first with git branch.

Phase 2 — Azure Deployment
CI/CD pipeline: push to main triggers build and deploy.

Wired up GitHub Actions. Every push: TypeScript compiles, Docker image builds, pushes to GHCR, deploys to Azure Container Instances. You push. It ships. You don't touch Azure manually. That's the goal. The pipeline failed on the first run.

ERROR: Incorrectly formatted secure environment settings.
Argument values should be in the format a=b c=d

broke Environment variables in the az container create command were split across lines. YAML looked fine. Azure CLI did not agree.

learned Azure CLI expects environment variables as space-separated key=value pairs on a single line. Multi-line YAML wrapping breaks it silently until the error surfaces at deploy time.

Phase 3–4 — GitHub Commits + Browser Portal
Artefacts commit to GitHub. Users access via browser.

Generated artefacts now write to /artefacts/{type}/ in the repo. Added a browser portal: enter topic, click generate, see the artefact, link to GitHub. Deploy succeeded. Container started. App never responded.

# start.sh was running:
node dist/index.js

# compiled output actually lived at:
dist/app/app.js

broke One file owned the entry point path. Another file referenced a different path. The container started clean and died immediately with no useful error.

learned One source of truth. package.json owns the correct path via npm start. Everything else defers to it. This is the pattern for every path reference in a project.

Mid 2026 — Expanding the practice
The pattern established. Scaling to more tools.

VAF Agentic Architect live at velocityarchitectureframework.com. The mode is proven: define the outcome, give Claude the brief, read and review the code, ship. Applied to VSF Match, EA Artefact Generator, four certification study sites, a trading dashboard. All built the same way.

May 2026
Two certifications completed. 12 repos live.

Completed CCA-F (Claude 101) and GH-600 (Claude Code in Action) back to back. Studied for both while actively building with the tools they certify. The theory was already practice. Zero coding experience at the start. Twelve repos now.
